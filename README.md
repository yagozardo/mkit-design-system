# MKIT Design System Documentation Site

A proof-of-concept documentation site for the Winsupply Marketing IT UX Design System, auto-generated from Figma using the Figma REST API and a Python static site generator — no build pipeline, no framework.

**Live site:** https://yagozardo.github.io/mkit-design-system/

---

## Background

Until this project, the design system existed entirely inside Figma — components, tokens, and usage notes were only accessible to designers with Figma access. This PoC explores what it looks like to surface that information as a proper documentation site: browsable by developers, shareable via URL, and kept in sync with Figma through a repeatable extraction workflow.

---

## Tech stack

- **Static HTML** — no framework, no npm, no build pipeline
- **Tailwind CSS** — via Play CDN (no installation required)
- **CSS custom properties** — all design tokens exposed as `--token-name` variables in `assets/tokens.css`
- **Python 3** — `build.py` generates all HTML from markdown content + JSON token data
- **Figma REST API** — used to extract tokens, component metadata, and export preview images

## Running locally

```bash
# Serve the site at http://localhost:8765
python3 -m http.server 8765
```

## Rebuilding the site

```bash
# Local (no base path prefix)
python3 build.py

# Production (GitHub Pages subpath)
SITE_BASE=/mkit-design-system python3 build.py
```

## Re-fetching component images from Figma

```bash
python3 fetch_images.py
```

Images are saved to `assets/images/` and cached — existing files are skipped.

## Deploying

```bash
SITE_BASE=/mkit-design-system python3 build.py
git add -A
git commit -m "rebuild"
git push
```

GitHub Pages rebuilds automatically on push to `main`.

## Known limitations (PoC scope)

- **Content is AI-generated** — component descriptions, usage guidelines, and accessibility notes were generated by Claude Code from Figma metadata and should be reviewed by the design team before treating as authoritative
- **No live token sync** — tokens are extracted on demand; a Figma webhook + CI/CD pipeline would be needed for automatic sync
- **Shadows pending** — L1–L6 elevation shadow tokens were not extractable via the REST API and remain as placeholders
- **Popover image** — the Popover component has no component set in Figma; its preview image is a placeholder
